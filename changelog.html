v2.4.0 Latest
June 27, 2024
Svelte 5 runes, Drizzle ORM integration, Hono v4 upgrade
Added
Svelte 5 runes mode enabled by default in the sveltekit template — $state, $derived, and $effect used throughout the starter.
Drizzle ORM integration as an alternative to Prisma — pass --orm drizzle to the CLI to use the new Drizzle template with migration tooling included.
New --features flag accepting a comma-separated list: --features auth,db,stripe,email for single-command full-stack setup.
OpenTelemetry tracing scaffold added to the Node.js API template. Exports to OTLP-compatible endpoints out of the box.
Changed
Hono upgraded from v3.12 to v4.0 in the Node.js API template. Route definitions updated to use the new Hono class constructor pattern.
Tailwind CSS configuration now uses the v4 CSS-first model — tailwind.config.ts no longer generated; all customisation lives in globals.css.
Bun dependency resolution updated to use bun install --frozen-lockfile in CI workflows to prevent accidental version drift.
Fixed
Auth.js session cookie sameSite attribute defaulting to lax instead of strict in the Next.js template middleware.
Path alias @/ not resolving correctly in Vite projects when output directory was set to a non-default value.
Docker multi-stage build failing on Alpine base images when node-gyp dependencies were present in the lockfile.
vercel.json — new rewrite rules for parallel routes
{
- "rewrites": [{ "source": "/", "destination": "/dashboard.html" }]
+ "buildCommand": null,
+ "outputDirectory": ".",
+ "framework": null,
+ "rewrites": [{ "source": "/", "destination": "/dashboard.html" }]
}
v2.3.0
May 14, 2024
TanStack Router, React 19 RC support, performance improvements
Added
TanStack Router v1 added as an option for the React SPA template, replacing the previous React Router v6 default. Pass --router tanstack or --router react-router.
React 19 RC support across all React-based templates. Concurrent features enabled by default with createRoot.
Storybook 8 pre-configured in the React SPA template with Chromatic CI integration and auto-generated stories for all scaffold components.
Fixed
Scaffold time regression introduced in v2.2.0 — parallel template download restored, bringing average time back below 1.5 seconds.
quickforge list command not displaying template descriptions when run in a CI environment with no TTY.
v2.2.1
April 30, 2024
Patch — Auth.js session bug and ESLint flat config fix
Fixed
Auth.js v5 session token not persisting across hard refresh when trustHost: true was not set in the auth config — added automatically for all environments.
ESLint flat config (eslint.config.mjs) failing on Node.js 18 due to missing --experimental-vm-modules flag in the lint script.
Prisma schema not found error when scaffold was run from a directory path containing spaces on Windows.
v2.2.0
April 8, 2024
Monorepo support, Turborepo pipeline, multi-app scaffold
Added
Monorepo mode: quickforge create --monorepo scaffolds a Turborepo workspace with a Next.js web app and a shared packages directory pre-configured.
quickforge add command for adding a new app or package to an existing QuickForge monorepo without re-scaffolding.
Shared @repo/ui and @repo/config packages generated with sensible defaults — Tailwind config, TypeScript config, and ESLint config all shared.
Changed
Default package manager switched from npm to Bun across all templates. npm and pnpm still supported via --pm npm and --pm pnpm.
v2.1.0
March 5, 2024
Docker output, GitHub Actions CI, Resend email integration
Added
Multi-stage Dockerfile generated for all templates — Node.js 22 Alpine base, non-root user, and .dockerignore excluding dev dependencies and local builds.
GitHub Actions workflow file for test, build, and deploy-to-Vercel on pull request merge, including environment-scoped secrets injection.
Resend email integration: pass --features email to include a pre-built transactional email module using Resend with React Email templates.
v2.0.0
February 1, 2024
Complete rewrite — plugin architecture, template registry, CLI v2
Added
Plugin architecture: publish a QuickForge plugin to npm with the quickforge-plugin-* namespace and it becomes available via --plugins at scaffold time.
Remote template registry at registry.quickforge.dev — community templates fetched at runtime, no local caching of stale templates.
Interactive CLI mode: running npx quickforge without arguments launches the step-by-step scaffold wizard.
Removed
v1 config file format (.quickforgerc) removed. Migrate to quickforge.config.ts using the migration guide.
Legacy --template flag removed in favour of --stack. The old flag prints a migration hint and exits.
