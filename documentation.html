Getting started / Introduction

Introduction

QuickForge is a production-oriented scaffolding engine for web applications. It generates fully configured, deployment-ready projects from opinionated templates — with Auth, database, styling, and CI/CD pre-wired.

What is QuickForge

Most scaffolding tools give you a project skeleton. QuickForge gives you a working application. Every template starts from a state where npm run dev works on the first try, npm run build produces a deployable artefact, and the CI pipeline passes before you write a single line of business logic.

The engine pulls templates from a remote registry at scaffold time, so you always get the latest version of each dependency and configuration pattern rather than a stale snapshot baked into an npm package.

QuickForge requires Node.js 18+ and Bun 1.0+. If you prefer npm or pnpm, pass --pm npm or --pm pnpm to any command.

Design philosophy

QuickForge is built on three principles:

Opinionated by default, configurable when it matters

The defaults are chosen to reflect production best practices — TypeScript strict mode, ESLint flat config, Prettier, Husky pre-commit hooks. You opt out of defaults rather than opt in to them.

No abstraction on top of your stack

QuickForge writes standard Next.js, React, Hono, or SvelteKit code. There is no QuickForge runtime, no QuickForge component library, and no QuickForge-specific APIs. The generated project is a plain Next.js project the day after scaffolding.

Reproducible across machines and time

Lockfiles are committed, Docker builds are deterministic, and CI uses frozen installs. A project scaffolded today should build identically six months from now on a different machine.

Installation

You do not need to install QuickForge globally. Use npx or bunx to run it at scaffold time:

bash
# Using npx (no global install required) npx quickforge create my-app # Or using bunx bunx quickforge create my-app # Install globally if you scaffold frequently npm install -g quickforge bun add -g quickforge

Quickstart — Next.js 15 in 60 seconds

1
Scaffold the project
Run the create command. QuickForge will fetch the template, resolve dependencies with Bun, and write the project to disk.
2
Configure environment variables
Copy .env.example to .env.local and fill in the required values. The scaffold includes a typed env schema that will fail fast at startup if anything is missing.
3
Run the database migration
Run bun run db:push to apply the baseline Prisma schema to your database. The connection string comes from DATABASE_URL in your .env file.
4
Start the dev server
Run bun run dev to start Next.js with Turbopack. The app is available at localhost:3000 with hot module replacement enabled.
bash
# Scaffold with auth and database pre-configured npx quickforge create my-app \ --stack next-js-v15 \ --auth github,google \ --db postgresql # Navigate and configure cd my-app cp .env.example .env.local # edit .env.local with your credentials # Push schema and start bun run db:push bun run dev

CLI flags reference

Flag Default Description
--stack next-js-v15 Template to scaffold. One of next-js-v15, react-spa, node-express, sveltekit.
--auth none Comma-separated OAuth providers to configure. e.g. github,google,discord.
--db none Database type for Prisma or Drizzle schema. One of postgresql, mysql, sqlite, mongodb.
--orm prisma ORM to use. One of prisma, drizzle.
--pm bun Package manager. One of bun, pnpm, npm.
--features none Additional integrations. Comma-separated: stripe, email, analytics.
--monorepo false Scaffold a Turborepo monorepo instead of a single app.
--no-git false Skip git initialisation. Useful when adding to an existing repository.
All flags are optional. Running npx quickforge create without flags launches the interactive wizard that guides you through each choice.

Project structure — Next.js 15

The generated Next.js project follows the App Router conventions with a predictable directory layout:

tree
my-app/ ├── src/ │ ├── app/ # App Router pages and layouts │ │ ├── (auth)/ # Route group — auth-gated pages │ │ │ └── dashboard/ │ │ ├── api/ # Route handlers │ │ │ └── auth/[...nextauth]/ │ │ ├── layout.tsx │ │ └── page.tsx │ ├── components/ # Shared UI components │ ├── lib/ # Utility functions and clients │ │ ├── auth.ts # Auth.js config │ │ ├── db.ts # Prisma client singleton │ │ └── env.ts # Typed env schema (t3-env) │ └── middleware.ts # Auth middleware ├── prisma/ │ └── schema.prisma ├── .env.example ├── Dockerfile ├── next.config.ts └── tsconfig.json

CLI reference / quickforge create

quickforge create

Scaffold a new project from a QuickForge template. The create command fetches the chosen template from the registry, resolves all dependencies, and writes the project to disk.

Syntax

bash
npx quickforge create [project-name] [flags]

Examples

bash
# Interactive mode npx quickforge create # Minimal scaffold — no auth, no database npx quickforge create my-site --stack next-js-v15 # Full-stack with PostgreSQL and GitHub OAuth npx quickforge create my-saas \ --stack next-js-v15 \ --auth github \ --db postgresql \ --features stripe,email # SvelteKit with SQLite for a lightweight project npx quickforge create my-blog \ --stack sveltekit \ --db sqlite \ --pm pnpm
